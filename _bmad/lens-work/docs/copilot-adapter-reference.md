# Copilot Adapter — lens-work v3

This directory contains the thin VS Code Copilot adapter that wires the `@lens` agent into the IDE. It references the lens-work module by path — it never duplicates module content.

## Structure

```
.github/
├── copilot-instructions.md         # References BMAD framework + lens-work module
├── lens-work-instructions.md       # lens-work-specific instructions for Copilot
├── agents/
│   └── bmad-agent-lens-work-lens.agent.md  # Thin wrapper → @lens agent in release module
├── prompts/
│   ├── lens-work.onboard.prompt.md
├──   ├── lens-work.create-initiative.prompt.md
│   ├── lens-work.preplan.prompt.md
│   ├── lens-work.businessplan.prompt.md
│   ├── lens-work.techplan.prompt.md
│   ├── lens-work.devproposal.prompt.md
│   ├── lens-work.sprintplan.prompt.md
│   ├── lens-work.dev.prompt.md
│   ├── lens-work.status.prompt.md
│   ├── lens-work.discover.prompt.md
│   └── ... (one stub per module command)
└── skills/
    ├── lens-work-git-state/SKILL.md
    ├── lens-work-git-orchestration/SKILL.md
    ├── lens-work-constitution/SKILL.md
    ├── lens-work-sensing/SKILL.md
    └── lens-work-checklist/SKILL.md
```

## Principles

- **Thin adapter** — references only, never copies or duplicates module content
- **Path references** — skills and workflows are referenced by path to the release module
- **Auto-propagation** — changes to module skills/workflows propagate automatically
- **Domain 3** — `@lens` does NOT modify `.github/` during initiative work
- **Write frequency** — RARELY, only on module version update
- **Prompts: lens-work only** — `.github/prompts/` contains exclusively `lens-work*.prompt.md` stubs; the pipeline enforces this constraint

## Generated vs. Authored Files

All `.github/` adapter files are **generated by the CI/CD pipeline** (`promote-to-release.yml`) via the `_module-installer/installer.js`. Source of truth for these files is the installer logic in `bmad.lens.src/lens.core/_bmad/lens-work/_module-installer/`.

Manual edits to source stub files (`.github/agents/`, `.github/skills/`) are overlaid on top of installer output by the pipeline, so source edits take precedence.
